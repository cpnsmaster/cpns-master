# CPNS Master — Project Principles

> Belajar lebih terarah. Latihan lebih cerdas. Lulus lebih siap.

## 1. User First

Setiap fitur CPNS Master harus memberikan manfaat nyata bagi proses belajar pengguna.

Pengalaman pengguna harus sederhana, jelas, nyaman, dan membantu pengguna mencapai tujuan belajarnya.

---

## 2. Content Is Data

Materi, soal, pembahasan, kategori, paket ujian, pengumuman, dan konten lainnya harus dikelola sebagai data.

Konten tidak boleh di-hardcode di dalam frontend jika konten tersebut berpotensi diubah oleh admin.

---

## 3. Simple Over Complex

Jika dua solusi dapat memberikan hasil yang sama, kita memilih solusi yang lebih sederhana.

Kompleksitas hanya boleh ditambahkan jika memberikan manfaat yang jelas.

---

## 4. Quality Over Quantity

Kualitas materi dan soal lebih penting daripada sekadar mengejar jumlah.

Seribu soal yang buruk tidak lebih baik daripada soal yang lebih sedikit tetapi memiliki kualitas, pembahasan, dan relevansi yang baik.

---

## 5. Scalable by Design

Setiap keputusan arsitektur harus mempertimbangkan kemungkinan perkembangan proyek di masa depan.

Namun, skalabilitas tidak boleh menjadi alasan untuk membuat sistem menjadi terlalu kompleks sejak awal.

---

## 6. Security by Design

Keamanan harus dipertimbangkan sejak awal pengembangan.

Meliputi:

- Authentication
- Authorization
- Row Level Security
- Validasi input
- Proteksi data pengguna
- Pengelolaan secret
- Keamanan API

---

## 7. Admin Empowerment

Admin harus dapat mengelola konten sebanyak mungkin melalui CMS tanpa harus mengubah source code atau melakukan deployment ulang.

---

## 8. Consistency

Konsistensi harus dijaga pada:

- UI
- UX
- Penamaan file
- Penamaan variabel
- Struktur folder
- Database
- API
- Dokumentasi

---

## 9. Learn Before Build

Setiap fitur harus memiliki tujuan yang jelas sebelum diimplementasikan.

Kita tidak membangun fitur hanya karena fitur tersebut terlihat menarik.

---

## 10. AI-Assisted, Human-Reviewed

AI dapat membantu dalam proses:

- Perencanaan
- Coding
- Dokumentasi
- Testing
- Debugging
- Review

Namun, keputusan akhir tetap berada pada manusia.

Semua kode yang masuk ke project harus dipahami dan direview sebelum digunakan.

---

## 11. Build for Real Users

CPNS Master harus dikembangkan dengan mempertimbangkan pengguna nyata.

Setiap fitur harus menjawab pertanyaan:

> Apakah fitur ini benar-benar membantu seseorang belajar lebih baik?

---

## 12. Continuous Improvement

CPNS Master akan terus berkembang.

Kita tidak harus membuat semuanya sempurna sejak versi pertama.

Namun, setiap versi harus lebih baik daripada versi sebelumnya.

---

# Core Philosophy

> **Learn deeply. Practice consistently. Improve continuously.**

---

# Product Motto

> **Belajar lebih terarah. Latihan lebih cerdas. Lulus lebih siap.**
